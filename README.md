KidLock - Privacy Policy (Gizlilik Politikası)
Last Updated: June 12, 2026

1. Information Collection and Use
KidLock ("we", "our", or "us") is developed as a remote parental control application. To perform its core functionality—specifically, blocking target applications defined by the parent—the application requires the activation of the Android AccessibilityService API.

2. AccessibilityService API Disclosure
Purpose: The AccessibilityService is strictly used to monitor package changes in the foreground. This allows the app to detect if a restricted application is opened and instantly launch the secure overlay lock screen.
Data Storage & Privacy: KidLock DOES NOT collect, harvest, log, or transmit any data processed through the Accessibility Service. No keystrokes, personal details, or app habits are sent to external servers or Firebase Realtime Database. All analysis is performed strictly on-device (locally).
3. Firebase Realtime Database Synchronization
The application uses Firebase Realtime Database solely to synchronize configuration settings (such as the list of application package names to be locked and the global lock state) between the parent's device and the child's device. No user content or private tracking data is stored on Firebase.

4. Data Sharing
We do not sell, trade, or otherwise transfer your private data to outside parties.
